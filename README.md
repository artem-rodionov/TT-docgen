# Генератор документов для шрифтовых проектов

Десктопное приложение для автоматической генерации пакета юридических документов (служебное задание, совместное заявление авторов, акты приёма-передачи) по данным из REST API компании, Excel-таблицы и `.glyphs`-файла шрифта.

---

## 📋 Функциональные возможности

- Загрузка списка проектов через REST API.
- Получение данных о проекте (даты, авторы, их роли) из API.
- Сопоставление имён авторов из API с реальными ФИО через маппинг в Excel.
- Чтение конфиденциальных данных сотрудников (паспорт, ИНН, должность) из Excel-таблицы.
- Парсинг `.glyphs`-файла – извлечение названия, версии и списка начертаний.
- Генерация трёх типов документов в формате `.docx` по шаблонам:
  - **Служебное задание** на создание/обновление шрифта.
  - **Совместное заявление соавторов** о передаче прав.
  - **Акты приёма-передачи** для каждого автора.
- Графический интерфейс на PySide6:
  - выбор проекта, дат, руководителя, стиля шрифта;
  - переключение типа работы (создание / обновление);
  - выбор генерируемых документов;
  - индикация прогресса;
  - диалог настроек (API-токен, пути к шаблонам и папке вывода).
- Кроссплатформенность (разработка на Ubuntu, сборка для macOS).

- Python **3.13**.

- Операционная система: Ubuntu, macOS. Windows в процессе разработки.

## ⚙️ Установка и настройка окружения

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/your-username/docgen.git
cd docgen
```

### 2. Установите uv

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

### 3. Создайте виртуальное окружение и установите зависимости

```bash
uv sync
```

Эта команда создаст `.venv` и установит все зависимости из `pyproject.toml` / `uv.lock`.

### 4. Активируйте виртуальное окружение

  ```bash
  source .venv/bin/activate
```

---

## 🚀 Запуск приложения

Или из корня проекта:

```bash
uv run python -m docgen.example
```

При первом запуске откроется окно настроек, где нужно указать:

- **API-токен** – для доступа к REST API (получить у администратора).
- **Путь к Excel-таблице** с данными о работниках (формат описан ниже).
- **Путь к шаблонам** `.docx` для задания, заявления и акта.
- **Папка для сохранения** сгенерированных документов.

Настройки сохраняются в `settings.json` в директории приложения.

## 📄 Формат Excel-таблицы

Приложение ожидает таблицу с двумя листами:

1. **Первый лист** – данные о работниках.  
   Столбцы (индексы 0..14):
   - `0` – подразделение
   - `1` – должность
   - `2` – **ФИО** (ключ для поиска)
   - `3` – дата рождения
   - `4` – тип документа (паспорт)
   - `5` – код подразделения
   - `6` – орган, выдавший паспорт
   - `7` – дата выдачи
   - `8` – номер паспорта
   - `9` – серия паспорта
   - `10` – адрес регистрации
   - `11` – ИНН
   - `12` – банк
   - `13` – расчётный счёт
   - `14` – БИК

2. **Второй лист** – маппинг имён (соответствие между именем в API и реальным ФИО).  
   Столбцы:
   - `0` – имя из API
   - `1` – реальное ФИО


## 📦 Сборка исполняемого файла (macOS)

Для сборки в `.app` используется PyInstaller. В репозитории уже настроен GitHub Actions (см. `.github/workflows/build.yml`), который автоматически собирает приложение при создании тега `v*`.

Локальная сборка:

```bash
uv run pyinstaller --windowed --onedir --name DocGen --hidden-import openstep_plist.util --hidden-import openstep_plist.parser src/docgen/example.py
```

Готовый бандл появится в `dist/DocGen.app`.

Для других ОС измените параметры PyInstaller.

## 📝 Примечания

- При запуске без настроек автоматически откроется диалог настроек.
- Все даты подставляются в формате, соответствующем шаблонам (день с ведущим нулём, название месяца в родительном падеже).
- В шаблонах используются теги `{{ variable }}` и циклы `{% for ... %}`, поддерживаемые библиотекой `docxtpl`.
- Для работы с `.glyphs` требуется библиотека `glyphsLib` (устанавливается автоматически).
